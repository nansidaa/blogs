    Vite第十九章：vite常用插件之vite-plugin-mock：🔥

    南思 2024/4/15 Vite学习笔记

    📒 http://mockjs.com/examples.html//mock.js文档

    📒 https://www.npmjs.com/package/vite-plugin-mock// vite-plugins-mock;npm 文档

    📒 mock 数据：就是模拟数据；

    前后端一般都是并行开发；
    mock 数据 去做你前端的工作

    📒 1. 简单方式: 直接去写死一两个数据 方便调试 -缺陷： - 没法做海量数据测试 - 没法获得一些标准数据 - 没法去感知 http 的异常。2.mockjs: 模拟海量数据的， vite-plugin-mock 的依赖项就是 mockjs 依赖项跑完后 在 config 里的 plugin 里将 引入的 vite-plugin-mock 加进去，然后在根目录下创建 mock 文件（他会自动找到这个 mock 文件）

    📒 mock 文件：

    const mockJS = require("mockjs");//commonjs引用mock包
const userList = mockJS.mock({ //mock里面必须是一个模板
  "data|100": [{//将 生成 100 条data数据 每个都是一个对象
    name: "@name", // @name mock语法：表示生成不同的中文名
    ename: mockJS.Random.last(), //@last 生成不同的英文名
    'age|11-99':1,//年龄
    address:'@county(true)',//随机地址
    email:'@email',//随机邮箱
    isMale:'@boolean',//随机性别
    createTime:'@datetime',//创建时间
    phone:/^1[34578]\d{9}$/,//随机电话号码
    "id|+1": 1, // 每个id 自增+1
    time: "@time",//生成不同的时间
    date: "@date"//生成不同的日期
    avatar() {//用户头像
    return Mock.Random.image('100×100',Mock.Random.color(),'#757575','png',this.nickName)
    }
  }]
})
module.exports = [//commonjs到处这个包
  {
    method: "post",
    url: "/api/users",
    response: ({ body }) => {
      // body -> 请求体
      // page pageSize body
      return {
        code: 200,
        msg: "success",
        data: userList
      };
    }
  },

]
